### Blocker 1 (rev 2): P05a flat-grep declaration checks replaced by semantic export-graph proof

**Original problem**: The P05a verification commands used flat greps on
`dist/index.d.ts` (`grep -q "createAgenticLoop"`, `grep -c 'export.*AgentClient'`,
etc.) to prove denied-name absence and curated-name presence. The package
root barrel (`packages/agents/index.ts`) is a bare
`export * from './src/index.js'`, so the emitted `dist/index.d.ts` is a single
export-star line with no literal curated-name substrings — the flat greps
failed because the names are reachable only through the transitive
export-star chain, not as literal text.

**Prior failed fix (rev 1 — REJECTED by verifier)**: A module-level JSDoc was
added to `packages/agents/index.ts` listing the curated names. TypeScript
preserves leading JSDoc into the emitted declaration, so the names appeared
as comment text in `dist/index.d.ts`. This made the flat greps pass but was
correctly rejected by the P05a verifier: comment text is NOT semantic proof
of API exports. Comments can name anything without the export graph actually
providing it.

**Correct fix (rev 2 — applied)**: The JSDoc-comment hack was removed from
`packages/agents/index.ts` (restored to the clean bare
`export * from './src/index.js'` barrel matching the committed HEAD state).
The P05a plan verification commands were updated to use the real
`apiSurfaceParser.mjs` (`parseExportedNames`) over the built
`dist/index.d.ts`. This resolves the export-star graph recursively and
asserts against the resolved exported-name SET:
- Denied names (`AgentClient`, `CoreToolScheduler`, `AgenticLoop`) are absent
  from the resolved set.
- Curated names (`createAgentClient`, `createToolScheduler`,
  `createTaskToolRegistration`, `createAgenticLoop`, `AgenticLoopRunner`,
  `AgenticLoopEvent`, `AgenticLoopMessage`) are present in the resolved set.

This is the same parser the standalone `lint:agents-api-surface` script uses
to build its report — it is not fooled by comment text because it uses the
TypeScript compiler API to parse the AST, not regex over file content.

```
$ cat packages/agents/index.ts
export * from './src/index.js';
(plus license/plan header — no JSDoc API-name hack)

$ node --input-type=module -e "..." (parseExportedNames over dist/index.d.ts)
```
