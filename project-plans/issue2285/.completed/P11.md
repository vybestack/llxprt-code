### Seven Characterization Suites

1. **Dispatch branch selection** (2 tests) — given `config.isInteractive()`
   true/false, the real `dispatchInteractiveOrNonInteractive` selects the
   interactive (`createForegroundAgent`) or non-interactive (`runNonInteractive`)
   branch. Asserts `dispatchTrace` contents (which path executed), NOT just that
   a seam was called.

2. **SIGINT handler installation/disposal** (3 tests) —
   `installNonInteractiveSigintHandler` installs a handler on `process`,
   writes "Cancelled." to stderr, calls `process.exit(130)` via the safe
   sentinel. Disposer removes the handler (listener count restored).
   Once-only guard prevents double-exit on repeated SIGINT.

3. **Output flush ordering** (3 tests) —
   `initializeOutputListenersAndFlush` installs Output + ConsoleLog listeners
   when none exist, drains backlogs, and does NOT install duplicates when a
   listener already exists. ConsoleLog routing verified via payload type/content.

4. **Process lifecycle / unhandled rejection handling** (3 tests) —
   `setupUnhandledRejectionHandler` installs an `unhandledRejection` listener
   on process; invoking it emits `LogError` app event containing the rejection
   reason and "Unhandled Promise Rejection" text. Disposer removes the handler.

5. **Piped prompt driving / non-interactive path** (3 tests) —
   `dispatchInteractiveOrNonInteractive` with `hasPipedInput=true` consumes
   stdin via `readStdinData` and reaches the runner; empty stdin + empty prompt
   exits 1 without reaching the runner; `hasPipedInput=false` with a prompt
   reaches the runner without calling `readStdinData`.

6. **Terminal/mouse cleanup** (3 tests) — `startInteractiveUI` registers
   `process.on('exit')` handlers for terminal protocol restoration; with mouse
   events enabled, additional exit handlers are registered (≥2 total). Ink
   `render` is called with the AppWrapper element tree (recording fake captures
   the call).

7. **Non-interactive error output** (6 tests) — `formatNonInteractiveError`
   delegates to `parseAndFormatApiError` for all inputs: plain Error and
   TypeError produce `[API Error: <message>]`; objects, numbers, null, and
   undefined produce `[API Error: An unknown error occurred.]`.

### Safe Seams Used (no mock theater)

- `installSafeProcessExit()` — `process.exit` replaced with sentinel-throwing
  stub; assertions verify the exit code (130 for SIGINT, 1 for no-input, 0 for
  successful non-interactive exit).
- `installCapturedStdio()` — `process.stderr.write`/`process.stdout.write`
  captured into string buffers; assertions verify stderr content ("Cancelled.").
- `installListenerCapture()` — `process.on`/`process.off` captured by event
  name; assertions verify listener counts and invoke captured handlers to test
  their observable effects.
- Recording Ink render fake — captures render calls without a real TTY;
  assertions verify render was invoked.
- Dependency mocks for heavyweight externals (`createForegroundAgent`,
  `runNonInteractive`, `validateNonInteractiveAuth`, `getStartupWarnings`,
  `getUserStartupWarnings`, `getCliVersion`, `checkForUpdates`,
  `handleAutoUpdate`) — the real dispatch code runs through these; assertions
  verify dispatch trace contents and observable effects.
