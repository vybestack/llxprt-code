## Implementation Sub-Step 2

### apiSurfaceParser.mjs

- Plain ESM `.mjs` (revision 3 finding 2) importable by both standalone Node
  scripts and the Vitest test without a TS loader.
- Uses the TypeScript compiler API via `createRequire('typescript')` (same
  pattern as `scripts/check-cli-import-boundary.mjs`).
- `parseExportedNames(declarationPath, visited?)`: recursively resolves
  `export * from '...'` by normalizing `.js` specifiers to `.d.ts` declaration
  files (revision 6 finding 3). Strips `.js`/`.mjs`/`.cjs`/`.ts`/`.tsx`/`.d.ts`
  suffixes and appends `.d.ts`, resolving relative to the current declaration
  file's directory. Handles all re-export forms: `export *`, `export { } from`,
  `export type { } from`. Extracts value exports (`export { X }`, `export class
  X`, `export function X`, `export const/let/var X`, `export enum X`) and
  type-only exports (`export type { X }`, `export interface X`, `export type X`).
  Cycle-safe via the `visited` Set.
- `DENIED_INTERNAL_NAMES`: frozen Set of `AgentClient`, `CoreToolScheduler`,
  `AgenticLoop`.
- `loadExpectedSurface(snapshotPath)`: reads the JSON snapshot into a Set.
- `API_SURFACE_REPORT_PATH`: `node_modules/.cache/agents-api-surface/report.json`
  relative to repo root (revision 4 finding 2).

### scripts/check-agents-api-surface.mjs (mechanism B1a)

## Remediation: P03a Alias Finding (local named-export alias names)

**Finding (P03a verifier, blocking):** `collectDirectExports` in
`apiSurfaceParser.mjs` recorded the local/original name (`X`) instead of the
exported name (`Y`) for LOCAL named exports of the form `export { X as Y }`.
The plan (REQ-002 — pseudocode api-surface-guard.md) explicitly requires
`export { X as Y }` records `Y`. The same defect affected the
re-export-from-module branch inside `collectDirectExports`, which recorded the
original name instead of the exported alias (the recursive `visit` in
`parseExportedNames` already recorded the correct alias, so re-export aliases
were not missing — but a spurious original name leaked).

**Reproduction (pre-fix):** against a temp declaration fixture
(`export { Internal as PublicAlias };` and
`export type { Hidden as PublicType } from './types.js';`) the parser produced
`Hidden,Internal,PublicType` — missing `PublicAlias`, leaking `Internal`/`Hidden`.

**Root cause:** both `collectDirectExports` named-export branches used
`el.propertyName ?? el.name`, which resolves to `el.propertyName` (the
local/original name) when an alias is present. The correct name for the public
surface is `el.name` (the exported alias seen by consumers). For unaliased
`export { X }`, `el.propertyName` is absent so `el.name === X` — unaffected.

**Fix:** changed both branches to record `el.name.text` (the exported name):
- Local named exports (`export { X as Y }`): now records `Y`.
- Re-export-from-module (`export { X as Y } from './m.js'`): now records `Y`
  (previously recorded `X`; the recursive `visit` already added `Y`).

**Files changed (remediation scope — P03-owned artifacts only):**
- `packages/agents/src/api/__tests__/apiSurfaceParser.mjs` — fixed both
  `collectDirectExports` named-export branches to record `el.name.text`.
