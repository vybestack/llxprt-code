## Proof Approach Fix (coordinator finding)

The prior P08 worker's proof script used `git worktree add --detach`, which
checks out clean HEAD. Because prior P03–P07 changes are currently
uncommitted in the working tree (33 entries: tracked-modified + untracked,
non-ignored), the clean-HEAD worktree did NOT contain those changes. The
proof therefore did not exercise the exact current source state plus the P09
migration — a material semantic gap.

The fix replaces the worktree approach with a disposable full-repo copy of
the CURRENT working tree, built from:

- `git ls-files -z` — tracked files at working-tree content (includes
  tracked-modified P03–P07 changes).
- `git ls-files --others --exclude-standard -z` — untracked, non-ignored
  files (includes untracked P03–P07 additions).

Both honor `.gitignore`, so `.git`, `node_modules`, dist/build outputs, and
ignored artifacts are excluded by git. The proof copies each listed file from
the real working tree into the temp copy (preserving relative path and
current content), runs `npm install` in the copy to create real workspace
links, applies the P09 changes to the COPY ONLY, then runs
`npm run build` + `npm run typecheck`. The copy is removed on exit via
exit/SIGINT/SIGTERM handlers.

This proves the proposed core interface resolves under the ACTUAL production
workspace typecheck against the exact current source state — the same
resolution P09 will exercise — BEFORE P09 touches production source.

Production interface declarations in the real working tree were NOT modified.
