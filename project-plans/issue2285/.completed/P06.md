## Characterization Proof Details

The proof (`boundary-checker-characterization-proof.mjs`) runs 5 assertions
that characterize the CURRENT (old) boundary checker behavior. Each assertion
is GREEN today (characterization — the proof passes by asserting the old
behavior exists):

1. **Symbol-level gap (the real gap)**: the old checker STILL flags a
   bare-root INTERNAL-symbol import (`AgentClient`) with the
   `agents-internal-symbol` classification — because `PUBLIC_AGENT_SYMBOLS`
   is still present. P07 removes `PUBLIC_AGENT_SYMBOLS`, after which bare root
   imports are allowed at the specifier level.

2. **Internals subpath (already caught, no false gap)**: the old checker
   ALREADY flags `@vybestack/llxprt-code-agents/internals.js` as a
   deep-import violation (`static-import` classification) — because the
   internals subpath is NOT in `PUBLIC_SUBPATHS_BY_PACKAGE` for the agents
   package. This is correct current behavior, not a gap.

3. **Deep source path**: the old checker flags
   `@vybestack/llxprt-code-agents/core/client.js` as a deep-import violation.

4. **Public symbol allowed**: the old checker ALLOWS a bare-root PUBLIC
   symbol import (`createAgent`) — documenting that `PUBLIC_AGENT_SYMBOLS`
   is selective.

5. **Namespace import**: the old checker flags a namespace import
   (`import * as ns`) from the bare agents root with the
   `agents-namespace-import` classification.

All assertions target exact violation literals/specifiers/classifications
(architect finding 9), not broad greps. Fixture files are generated via Node
(`mkdtempSync`/`writeFileSync`/`rmSync`), not sed/printf (revision 3 finding
18). The proof runs in a temp directory and never edits production source.
