# Disambiguation Export Audit (P05, architect finding 7)

Plan: PLAN-20260629-ISSUE2285.P05
Date: 2026-06-29

## Purpose

After removing `export * from './internals.js'` from
`packages/agents/src/index.ts`, the `export *` merge conflict that previously
required explicit disambiguation re-exports no longer exists for names already
re-exported by `./api/index.js`. This audit records the three-evidence trail
(snapshot + emitted declaration + api-barrel) for EACH disambiguation export
that existed in the pre-depollution root, and records the remove/retain
decision with concrete evidence.

No symbol was removed without this evidence trail.

## Evidence sources

1. **Snapshot evidence**: the depolluted root surface report generated by
   `npm run lint:agents-api-surface` after removing
   `export * from './internals.js'` and the disambiguation re-exports
   (`node_modules/.cache/agents-api-surface/report.json`, 156 names).
2. **Declaration evidence**: `npm run build --workspace @vybestack/llxprt-code-agents`,
   then grep the freshly emitted `dist/` declaration tree.
3. **api-barrel evidence**: grep `packages/agents/src/api/index.ts` to confirm
   whether the symbol is already re-exported from the api barrel.

## Per-symbol decisions

### `AgenticLoopMessage` — REMOVE (redundant)

- **Snapshot evidence**: present in the depolluted root surface report (151
  names). After removing the explicit
  `export type { AgenticLoopMessage } from './core/agenticLoop/types.js'`, the
  name survives via `export * from './api/index.js'`.
- **Declaration evidence**: `dist/src/core/agenticLoop/types.d.ts:127` declares
  `export type AgenticLoopMessage = PartListUnion;`.
  `dist/src/api/runtimeFactories.d.ts:72` re-exports it
  (`export type { ..., AgenticLoopMessage, ... }`).
  `dist/src/api/index.d.ts:20` re-exports it from `runtimeFactories.js`.
  The root `dist/src/index.d.ts` does `export * from './api/index.js'`, so
  `AgenticLoopMessage` reaches the root transitively.
- **api-barrel evidence**: `packages/agents/src/api/index.ts:61` contains
  `AgenticLoopMessage` in the `export type { ... } from './runtimeFactories.js'`
  block.
- **Same underlying type**: `runtimeFactories.ts` imports `AgenticLoopMessage`
  from `'../core/agenticLoop/index.js'` (the same module the explicit
  disambiguation re-export targeted). The root definition is byte-identical to
  the pre-depollution root definition. The explicit re-export is REDUNDANT.
- **Decision**: remove the explicit disambiguation re-export.

### `ApprovalHandler` — REMOVE (redundant)

- **Snapshot evidence**: present in the depolluted root surface report.
  After removing the explicit
  `export type { ApprovalHandler } from './core/agenticLoop/types.js'`, the
  name survives via `export * from './api/index.js'`.
- **Declaration evidence**: TWO declarations exist in the dist tree:
  - `dist/src/api/config-types.d.ts:150` — the PUBLIC API version:
    `(confirmation: { readonly confirmationId: string; ... }) => Promise<ToolConfirmationOutcome> | ToolConfirmationOutcome`.
  - `dist/src/core/agenticLoop/types.d.ts:67` — the low-level loop version:
    `(request: ToolConfirmationRequest) => Promise<ApprovalResult>`.
  `dist/src/api/index.d.ts:9` re-exports `ApprovalHandler` from
  `./config-types.js` (the public version). The root reaches the PUBLIC version
  via `export * from './api/index.js'`.
- **api-barrel evidence**: `packages/agents/src/api/index.ts:29` contains
  `ApprovalHandler` in the `export type { ... } from './config-types.js'` block.
- **Type-meaning note**: pre-depollution, the explicit disambiguation re-export
  forced the root `ApprovalHandler` to be the LOW-LEVEL agenticLoop version
  (overriding the api barrel's public config-types version via named-export
  precedence). After depollution, the root `ApprovalHandler` becomes the PUBLIC
  config-types version. This is an intentional semantic cleanup: the
  disambiguation existed only to resolve the `export *` merge between the
  internals barrel and the api barrel; with the internals barrel gone, the
  public API version is the correct root definition. No consumer imports
  `ApprovalHandler` from the agents root (verified by grep across all
  `packages/**/*.ts`). The low-level version remains available via the
  `./internals.js` subpath and via the curated alias `AgenticLoopApprovalHandler`
  in the api barrel.
- **Decision**: remove the explicit disambiguation re-export. The root
  `ApprovalHandler` resolves to the public API config-types version.

### `CompressionResult` — REMOVE (redundant)

- **Snapshot evidence**: present in the depolluted root surface report.
  After removing the explicit
  `export type { CompressionResult } from '@vybestack/llxprt-code-core/core/compression/types.js'`,
  the name survives via `export * from './api/index.js'`.
- **Declaration evidence**: `dist/src/api/agent.d.ts:61` declares
  `export interface CompressionResult { ... }`. `dist/src/api/index.d.ts` does
  `export * from './agent.js'`, so `CompressionResult` reaches the root
  transitively.
- **api-barrel evidence**: `packages/agents/src/api/index.ts` contains
  `export * from './agent.js'`, which transitively re-exports the
  `CompressionResult` interface declared in `api/agent.ts`.
- **Type-meaning note**: pre-depollution, the explicit disambiguation re-export
  forced the root `CompressionResult` to be the CORE compression types version
  (`@vybestack/llxprt-code-core/core/compression/types.js`), overriding the api
  barrel's `agent.ts` interface version via named-export precedence. After
  depollution, the root `CompressionResult` becomes the PUBLIC agent interface
  version. This is intentional: the disambiguation existed only to resolve the
  `export *` merge between the internals barrel and the api barrel; with the
  internals barrel gone, the public agent interface version is the correct root
  definition. No consumer imports `CompressionResult` from the agents root
  (verified by grep across all `packages/**/*.ts`). The core version remains
  available via the `./internals.js` subpath and via direct
  `@vybestack/llxprt-code-core/core/compression/types.js` imports.
- **Decision**: remove the explicit disambiguation re-export. The root
  `CompressionResult` resolves to the public API agent.ts interface version.
