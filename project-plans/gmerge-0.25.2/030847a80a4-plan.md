# Playbook: Export Chat History in /bug and Prefill GitHub Issue

**Upstream SHA:** `030847a80a4`
**Upstream Subject:** feat(cli): export chat history in /bug and prefill GitHub issue (#16115)
**Upstream Stats:** ~4 files, moderate insertions

## What Upstream Does

Upstream enhances the `/bug` slash command to:
1. Export the current chat history (conversation transcript) to a temporary file.
2. Include the transcript file path in the bug report URL or body so the user can attach it.
3. Create a `historyExportUtils.ts` utility module for sanitizing and exporting conversation history.
4. Prefill the GitHub issue template with system info and a reference to the exported history.

The utility sanitizes sensitive data (API keys, tokens) from the transcript before writing it, and formats the history as a readable Markdown document.

## Why REIMPLEMENT in LLxprt

1. LLxprt's `bugCommand.ts` (line 67) constructs a GitHub issue URL pointing to `github.com/vybestack/llxprt-code/issues/new` with `{title}` and `{info}` template variables — this is already LLxprt-branded and differs from upstream's Gemini URL.
2. `packages/cli/src/ui/utils/historyExportUtils.ts` does not exist yet and must be created from scratch for LLxprt's conversation model.
3. LLxprt's conversation history is managed through `useHistoryManager` and the `HistoryItem` type system in `packages/cli/src/ui/types.ts`, NOT upstream's format. The export utility must work with LLxprt's `HistoryItem[]` array.
4. Privacy: LLxprt must redact `LLXPRT_API_KEY`, provider API keys (OpenAI, Anthropic, etc.), and any credentials found in the transcript. The sanitizer must be provider-neutral, not just Gemini-focused.
5. The exported file should go to the OS temp directory and the path should be communicated to the user via the UI, not silently embedded.

 * Sanitizes and exports conversation history for bug reports.

/**
 * Patterns to redact from exported transcripts.
 * Provider-neutral: covers LLxprt, OpenAI, Anthropic, Google, and common secret patterns.
 */
const REDACTION_PATTERNS: Array<{ pattern: RegExp; replacement: string }> = [
  { pattern: /LLXPRT_API_KEY\s*[:=]\s*\S+/gi, replacement: 'LLXPRT_API_KEY=[REDACTED]' },
  { pattern: /OPENAI_API_KEY\s*[:=]\s*\S+/gi, replacement: 'OPENAI_API_KEY=[REDACTED]' },
  { pattern: /ANTHROPIC_API_KEY\s*[:=]\s*\S+/gi, replacement: 'ANTHROPIC_API_KEY=[REDACTED]' },
  { pattern: /GEMINI_API_KEY\s*[:=]\s*\S+/gi, replacement: 'GEMINI_API_KEY=[REDACTED]' },
  { pattern: /sk-[a-zA-Z0-9]{20,}/g, replacement: '[REDACTED_API_KEY]' },
  { pattern: /ghp_[a-zA-Z0-9]{36}/g, replacement: '[REDACTED_GH_TOKEN]' },
  { pattern: /AKIA[0-9A-Z]{16}/g, replacement: '[REDACTED_AWS_KEY]' },
  { pattern: /Bearer\s+[a-zA-Z0-9._-]{20,}/gi, replacement: 'Bearer [REDACTED]' },
];

/**
 * Sanitize text by redacting known secret patterns.
 */
export function sanitizeTranscript(text: string): string {
  let result = text;
  for (const { pattern, replacement } of REDACTION_PATTERNS) {
    result = result.replace(pattern, replacement);
  }
  return result;
}

## Execution Notes / Risks

- **Risk: History access from CommandContext.** The `CommandContext` in `types.ts` may not directly expose conversation history. If not, the implementer must check whether `context.ui` has a `getHistory()` method or whether history items need to be passed through a different mechanism. If history is not accessible, the export feature should degrade gracefully — export system info only with a note that history was not available.
- **Risk: URL length.** Do NOT embed the full transcript in the GitHub issue URL. URLs have practical limits (~8000 chars). The history is exported to a file, and the user is told to attach it.
- **Privacy:** The sanitizer must be conservative — redact anything that looks like a credential. The `REDACTION_PATTERNS` list should err on the side of over-redacting rather than leaking secrets.
- **Branding:** Use `llxprt-bug-report` in filenames, `LLxprt` in headers. GitHub URL is already `github.com/vybestack/llxprt-code`.
- **Do NOT** use upstream's `historyExportUtils.ts` verbatim — it references `Gemini` types and upstream's conversation model.
- **Do NOT** change the existing bug report URL template structure — the `{title}` and `{info}` replacements and the `bugCommandSettings.urlTemplate` override must continue to work.
