## Quality Assessment

### Behavioral Tests (not file-existence checks)
- Interface contract tests exercise method signatures through structural fakes with actual return values
- Registry contract tests construct real tool instances with injected services and verify observable output
- Key masking tests compute and verify actual masked strings
- Package boundary tests read and parse real package.json content

### Tests Would Fail If Implementation Were Broken
- If an interface method were removed or renamed, the structural fake would fail to compile (typecheck) and fail at runtime
- If package.json gained a forbidden dependency, the package-boundary test would detect it
- If forbidden imports appeared in production source, the ripgrep scan would detect them
- If ToolRegistry.register didn't store tools, listTools/getTool would return empty/undefined

### No Mock Theater
- No `vi.mock` calls
- No `NotYetImplemented` patterns
- No reverse testing (testing that wrong things fail)
- Structural fakes are used only for infrastructure dependencies (IToolHost, IShellExecutionService, etc.), not for the unit under test

### Forbidden Import Detection
- Uses ripgrep with `from` import statement matching
- Scans production source only (excludes `__tests__/`)
- Detects both package-qualified imports (`@vybestack/llxprt-code-core`) and relative path imports (`packages/core/src`)
