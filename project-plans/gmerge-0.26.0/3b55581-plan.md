# REIMPLEMENT Playbook: 3b55581 — Add experimental extension config setting

## Upstream Change Summary

This commit adds a new experimental setting `extensionConfig` (boolean, default `false`) that controls whether extension configuration features are enabled. When disabled:
- The `configure` command shows an error and exits
- Extension installation skips settings prompts and fetching
- Missing settings checks are bypassed

The setting requires a restart and is added to both the settings schema and documentation.

**Files changed upstream:**
- `docs/get-started/configuration.md` - Documentation for the new setting
- `packages/cli/src/commands/extensions/configure.ts` - Gate configure command on the setting
- `packages/cli/src/config/extension-manager-scope.test.ts` - Add setting to test fixtures
- `packages/cli/src/config/extension-manager.ts` - Gate settings fetching/prompting on the setting
- `packages/cli/src/config/extension.test.ts` - Add setting to test fixtures
- `packages/cli/src/config/extensions/extensionUpdates.test.ts` - Add setting to test fixtures
- `packages/cli/src/config/settingsSchema.ts` - Add schema definition
- `schemas/settings.schema.json` - JSON schema update (binary)

## Adaptation Plan

**ADAPT COMMIT (non-1:1 mapping)**

### 1. Add `experimental.extensionConfig` to settings schema

In `packages/cli/src/config/settingsSchema.ts`, add to the `experimental` section:

```typescript
extensionConfig: {
  type: 'boolean',
  default: false,
  description: 'Enable extension configuration features (settings prompting, fetching, and configure command).',
  requiresRestart: true,
},
```

### 2. Gate extension configure command

In `packages/cli/src/commands/extensions/config.ts`, check the setting at the start of the command handler. If `experimental.extensionConfig` is `false`, show an error message and return early.

### 3. Gate settings fetch/prompt in extension install/update

In `packages/cli/src/config/extension.ts`, where settings are currently fetched/prompted during install or update:
- Check `experimental.extensionConfig` before prompting for settings
- Check before fetching remote settings
- Skip missing-settings checks when the flag is `false`

### 4. Update tests

- In `packages/cli/src/config/extension.test.ts`, add test fixtures that include the new setting
- In `packages/cli/src/commands/extensions/config.test.ts`, add tests for:
  - Command blocked when `experimental.extensionConfig` is `false`
  - Command proceeds when `experimental.extensionConfig` is `true`
