- Legacy `new ProviderContentGenerator(providerManager, config)` path preserved as fallback
- No core→providers import was removed; injection path provides the escape hatch

### C-CB-04 (Runtime Provider Errors)
- `providerRuntimeContext.ts` now imports `MissingRuntimeProviderError` from `./errors/MissingRuntimeProviderError.js` instead of `../providers/errors.js`
