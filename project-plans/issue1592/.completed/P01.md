## Behavior Implemented

- Config no longer constructs AgentClient directly in core production code. AgentClient construction is supplied by ConfigParameters.agentClientFactory and is required at use time in initialize and initializeContentGeneratorConfig.
- schedulerSingleton no longer dynamically imports CoreToolScheduler. Scheduler construction is supplied by ConfigParameters.toolSchedulerFactory and required at scheduler creation time.
- toolRegistryFactory no longer imports tools/task.ts directly. P01 uses a temporary core-local defaultTaskToolRegistration module as the only allowed concrete TaskTool import until P03 deletes it and composition roots wire TaskTool from agents.
- CLI and a2a composition roots now provide AgentClient and CoreToolScheduler factories.
- Core contract modules provide structural AgentClientContract and ToolSchedulerContract surfaces for staying core consumers.
