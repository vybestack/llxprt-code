## Holistic Functionality Assessment

The implementation preserves the original runtime behavior while changing ownership boundaries. Core no longer constructs or imports concrete agent runtime classes; instead, core exposes structural contracts and receives factories from composition roots. CLI and A2A now import `@vybestack/llxprt-code-agents`, construct `AgentClient`/`CoreToolScheduler`, and register the `TaskTool` through `createTaskToolRegistration()`. Agents owns the concrete chat loop and subagent orchestration while continuing to use core-owned contracts, settings/runtime services, tools, policy, and shared type modules through explicit core subpath exports.

The data flow remains the same at runtime: CLI/A2A builds `Config`, injects concrete factories, `Config.initialize()` creates an agent client through the injected factory, and scheduler/tool registration flows through the injected scheduler and task-tool registration. The moved agents package tests, core tests, full workspace tests, build, typecheck, lint, lockfile check, and smoke tests all pass after the move.
