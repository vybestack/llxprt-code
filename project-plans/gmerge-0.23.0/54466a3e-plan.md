# Reimplement Plan: Hooks friendly names and descriptions (upstream 54466a3ea8)

## Upstream Change
Adds optional `name` and `description` fields to hook configuration. The `name` field serves as a unique identifier for hooks and is preferred over `command` for identification in UI and enable/disable operations. The `description` field provides human-readable documentation. Updates hook registry, planner, runner, and UI to use name for identification (falling back to command), and updates UI to display friendly names and descriptions.
