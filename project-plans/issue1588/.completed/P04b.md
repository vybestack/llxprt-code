## P04c Verification Remediation (P04b typecheck fix)

P04c verification failed because `npm run typecheck` exited nonzero across multiple workspaces. Root causes and fixes:

### Root Cause 1: Consumer tsconfig includes settings test files via `../settings/src/**/*.ts`

Core, CLI, and providers all include `../settings/src/**/*.ts` in their `include` patterns to resolve settings package imports via path aliases. However, the `exclude` patterns only used `**/*.test.ts` / `**/*.spec.ts` which are relative to the consumer package root and do NOT match `../settings/src/**/*.test.ts`. This caused TS6305 errors ("output file has not been built from source file") for all settings test files included in the program.

**Fix:** Added `../settings/src/**/*.test.ts` and `../settings/src/**/*.spec.ts` to the `exclude` arrays in:
- `packages/core/tsconfig.json`
- `packages/cli/tsconfig.json`
- `packages/providers/tsconfig.json`

Settings source resolution via `../settings/index.ts` and `../settings/src/**/*.ts` remains intact for production imports.

### Root Cause 2: `unknown` type assertions in SettingsService.test.ts

`SettingsServiceStub.exportForProfile()` returns `Promise<unknown>` and `getSettings()` returns `Promise<unknown>`. Test assertions that accessed properties on these values (e.g. `exported.defaultProvider`, `settings.providers`) produced TS18046 errors ("'x' is of type 'unknown'").

**Fix:** Added explicit type assertions (`as { defaultProvider: string; providers: ... }` etc.) to narrow the `unknown` values in the test file. This is purely a test typing fix — no production behavior was implemented.
