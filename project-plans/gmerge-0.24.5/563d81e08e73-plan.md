# Playbook: Add experimental in-CLI extension install and uninstall subcommands

**Upstream SHA:** `563d81e08e73`
**Upstream Subject:** Add experimental in-CLI extension install and uninstall subcommands
**Upstream Stats:** 6 files, 435 insertions

## What Upstream Does

Adds `/extensions install` and `/extensions uninstall` slash commands for managing extensions from within the interactive CLI. Extracts `inferInstallMetadata()` function from install logic to validate and categorize install sources (git URL, local path). Adds install/uninstall actions to `extensionsCommand.ts` with input validation (checks for command injection characters in source paths). Updates tests to mock `inferInstallMetadata` and verify command behavior. These commands are only available when `enableExtensionReloading` is true.

## Implementation Steps

1. **Extract inferInstallMetadata from extension-manager**:
   - Open `packages/cli/src/config/extension-manager.ts`
   - Find the install source validation logic in `installOrUpdateExtension` (around line 30-60 in upstream)
   - Extract to new function:
     ```typescript
     export async function inferInstallMetadata(
       source: string,
       args: {
         ref?: string;
         autoUpdate?: boolean;
         allowPreRelease?: boolean;
       } = {},
     ): Promise<ExtensionInstallMetadata>
     ```
   - Logic:
     - If source is URL (http://, https://, git@, sso://), return git metadata
     - Else, check for --ref or --autoUpdate and throw error (not applicable for local)
     - Try stat(source) to verify file exists
     - Return local metadata or throw "Install source not found"

2. **Update install.ts to use inferInstallMetadata**:
   - In `packages/cli/src/commands/extensions/install.ts`
   - Import `inferInstallMetadata` from extension-manager
   - Replace inline source validation with: `const installMetadata = await inferInstallMetadata(source, { ref, autoUpdate, allowPreRelease })`
   - Remove old stat() and URL checking logic

3. **Mock inferInstallMetadata in install.test.ts**:
   - Create hoisted mock: `const mockInferInstallMetadata: Mock<typeof inferInstallMetadata> = vi.hoisted(() => vi.fn())`
   - Add to vi.mock for extension-manager
   - In beforeEach, set default mock implementation:
     ```typescript
     mockInferInstallMetadata.mockImplementation(async (source, args) => {
       if (source.startsWith('http://') || source.startsWith('https://') || 
           source.startsWith('git@') || source.startsWith('sso://')) {
         return { source, type: 'git', ...args };
       }
       return { source, type: 'local' };
     });
     ```
   - Update "unknown source" test to mockRejectedValue
   - Clear mock in afterEach

4. **Add install subcommand to extensionsCommand.ts**:
   - Add `installAction` function:
     - Check extensionLoader is ExtensionManager instance
     - Parse source from args.trim()
     - Validate source is not empty (show usage message)
     - Validate source for safety (URL check OR no shell injection chars `/[;&|`'"]/.test(source)`)
     - Show "Installing extension from..." info message
     - Call `inferInstallMetadata(source)` and `installOrUpdateExtension(metadata)`
     - Show success or error message
   - Add `installCommand` object:
     ```typescript
     {
       name: 'install',
       description: 'Install an extension from a git repo or local path',
       kind: CommandKind.BUILT_IN,
       autoExecute: false,
       action: installAction,
     }
     ```
   - Add to conditionalCommands when enableExtensionReloading is true

5. **Add uninstall subcommand to extensionsCommand.ts**:
   - Add `uninstallAction` function:
