## What Upstream Does

Adds support for array-based `commandPrefix` in policy updates, allowing multiple shell command prefixes to be allowlisted in a single policy operation. Updates the policy engine to generate separate rules for each prefix with robust regex matching (using word boundaries to prevent partial matches like "git" matching "github"). Modifies `ShellToolInvocation.getPolicyUpdateOptions()` to extract root commands from chained commands and return them as an array, enabling both "Always" and "Always and Save" confirmation outcomes to properly allowlist compound commands.

### 3. Modify createPolicyUpdater in config.ts

**File:** `packages/core/src/policy/config.ts`
- Replace the single commandPrefix handling with array-based logic
- Convert commandPrefix to array if string: `const prefixes = Array.isArray(message.commandPrefix) ? message.commandPrefix : [message.commandPrefix]`
- Loop over prefixes to create multiple rules
- Update regex pattern to include word boundary: `new RegExp(\`"command":"${escapedPrefix}(?:[\\\\s"]|$)\`)`
- Move argsPattern creation into else block for non-commandPrefix cases

**Example implementation:**
```typescript
if (message.commandPrefix) {
  const prefixes = Array.isArray(message.commandPrefix) 
    ? message.commandPrefix 
    : [message.commandPrefix];
  
  for (const prefix of prefixes) {
    const escapedPrefix = prefix.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
    const argsPattern = new RegExp(`"command":"${escapedPrefix}(?:[\\s"]|$)`);
    
    rules.push({
      toolName: message.toolName,
      argsPattern,
      decision: PolicyDecision.ALLOW,
      priority: 100,
    });
  }
} else if (message.argsPattern) {
  // Handle non-commandPrefix cases
  rules.push({
    toolName: message.toolName,
    argsPattern: message.argsPattern,
    decision: PolicyDecision.ALLOW,
    priority: 100,
  });
}
```

### 4. Update ShellToolInvocation.getPolicyUpdateOptions()

**File:** `packages/core/src/tools/shell.ts`
- Import `getCommandRoots` and `stripShellWrapper` from `../utils/shell-utils.js` (already imported)
- For both `ProceedAlwaysAndSave` and `ProceedAlways` outcomes:
  - Strip shell wrapper from command: `const stripped = stripShellWrapper(this.params.command)`
  - Extract root commands: `const rootCommands = getCommandRoots(stripped)`
  - Deduplicate using Set: `const uniqueRoots = [...new Set(rootCommands)]`
  - Return array if length > 0, otherwise fallback to raw command

**Example implementation:**
```typescript
getPolicyUpdateOptions(): PolicyUpdateOptions | undefined {
  const outcome = this.confirmationOutcome;
  if (outcome?.action === 'ProceedAlwaysAndSave' || outcome?.action === 'ProceedAlways') {
    const stripped = stripShellWrapper(this.params.command);
    const rootCommands = getCommandRoots(stripped);
    const uniqueRoots = [...new Set(rootCommands)];
    
    return {
      commandPrefix: uniqueRoots.length > 0 ? uniqueRoots : stripped,
    };
  }
  return undefined;
}
```
