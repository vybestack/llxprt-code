**Upstream SHA:** `56092bd78205`
**Upstream Subject:** feat(hooks): Add a hooks.enabled setting (#15933)
**Upstream Stats:** 13 files, 68 insertions(+), 55 deletions(-)

## What Upstream Does

Introduces canonical `hooks.enabled` setting (default: false) to control hook execution, replacing scattered checks for `tools.enableHooks`. The new `getEnableHooks()` helper returns `(tools.enableHooks ?? true) && (hooks.enabled ?? false)`, ensuring both flags must be true for hooks to run. Updates all references to check `getEnableHooks()` instead of `tools.enableHooks`. Changes `tools.enableHooks` to an "experimental" toggle (default: true) that gates the entire hooks system infrastructure, while `hooks.enabled` is the user-facing on/off switch. Updates documentation, integration tests, and error messages.

## LLxprt Adaptation Strategy

LLxprt needs the same two-level gating:

1. **Setting**: Add `hooks.enabled` to settingsSchema.ts (default: false)
2. **Helper**: Add `getEnableHooks()` function to settingsSchema.ts
3. **Refactor**: Replace all `settings.tools?.enableHooks` checks with `getEnableHooks(settings)`
4. **Documentation**: Update configuration docs (if LLxprt has them)
5. **Tests**: Update integration tests to use `hooks.enabled` instead of `tools.enableHooks`
6. **Error messages**: Update user-facing messages to reference `hooks.enabled` instead of `tools.enableHooks`

Files to check: config.ts, extension-manager.ts, CLI entry points, hooks command, migrate command.

## Implementation Steps

1. **Modify `settingsSchema.ts`**:
   - In `hooks` properties, add:
     ```typescript
     enabled: {
       type: 'boolean',
       label: 'Enable Hooks',
       category: 'Advanced',
       requiresRestart: false,
       default: false,
       description: 'Canonical toggle for the hooks system. When disabled, no hooks will be executed.',
       showInDialog: false,
     }
     ```
   - Change `tools.enableHooks` description to: "Enables the hooks system experiment. When disabled, the hooks system is completely deactivated regardless of other settings."
   - Change `tools.enableHooks` default to `true`
   - Add helper function:
     ```typescript
     export function getEnableHooks(settings: Settings): boolean {
       return (settings.tools?.enableHooks ?? true) && (settings.hooks?.enabled ?? false);
     }
     ```

2. **Modify `config.ts` (CLI)**:
   - Import `getEnableHooks` from settingsSchema
   - Replace `settings.tools?.enableHooks` with `getEnableHooks(settings)` in parseArguments and loadCliConfig

3. **Modify `extension-manager.ts`**:

5. **Modify `hooksCommand.ts`**:

6. **Modify `migrate.ts`**:
