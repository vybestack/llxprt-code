## Extraction Details

### Code Moved (Lines 1691-1858 → tool-executor.ts)

**Extracted logic:**
- Before hook invocation (triggerBeforeToolHook)
- Hook blocking decision handling
- Tool input modification from hooks
- Live output callback setup
- PID tracking callback setup
- Tool invocation execution
- Cancellation handling (signal.aborted checks)
- After hook invocation (triggerAfterToolHook)
- Hook result application (systemMessage, suppressOutput)
- Error handling and propagation

**Adaptations for module boundaries:**
1. Changed direct scheduler method calls to callbacks:
   - `this.outputUpdateHandler` → `onLiveOutput` callback
   - `this.setPidInternal` → `onPid` callback
   - `this.bufferResult/bufferError/bufferCancelled` → return ToolExecutionResult
2. Changed status transitions:
   - Removed `this.setStatusInternal(callId, 'executing')` (done by caller)
3. Changed error handling:
   - Removed direct buffer calls, throw errors instead
   - Caller handles buffering based on result/exception

### Delegation Pattern (coreToolScheduler.ts)

**New launchToolExecution flow:**
1. Set status to 'executing' (before delegation)
2. Create ToolExecutor instance
3. Call toolExecutor.execute() with:
   - Scheduled call
   - Signal
   - onLiveOutput callback (wires to outputUpdateHandler + toolCalls update)
   - onPid callback (wires to setPidInternal)
4. On success: buffer result via bufferResult()
5. On error/cancellation: buffer via bufferError/bufferCancelled()
6. Call publishBufferedResults() with error recovery (#957)
