## What Upstream Does

Adds security warnings for project-level hooks through a `TrustedHooksManager` that tracks trusted hooks in `~/.gemini/trusted_hooks.json`. When hooks are loaded from project settings (`.gemini/settings.json`), the system checks if they've been trusted before. If untrusted hooks are detected in a trusted folder, a warning is displayed and the hooks are auto-trusted to prevent repeated warnings. The core changes: (1) adds `projectHooks` field to Config, (2) adds `trustedHooks.ts` module, (3) `hookRegistry` checks trust on initialization, (4) CLI passes project hooks separately from user hooks, (5) tests updated throughout.
