# Playbook: Fix Circular Dependency via tsconfig Paths

**Upstream SHA:** `881b026f2454a1ed549a58e441149b11f3a8a9e8`
**Upstream Subject:** fix(core): resolve circular dependency via tsconfig paths
**Upstream Stats:** 1 file, 5 insertions

## What Upstream Does

Adds a **tsconfig path mapping** to resolve circular dependencies within the `@google/gemini-cli-core` package. The fix:

1. Sets `baseUrl` to `.` (current directory)
2. Maps `@google/gemini-cli-core` to `./index.ts` within the core package

This allows files inside `packages/core/src/` to import from `@google/gemini-cli-core` and have TypeScript resolve it to the local `index.ts` instead of going through the compiled output, which can cause circular dependency issues.

## LLxprt Adaptation Strategy

LLxprt uses `@vybestack/llxprt-code-core` as the package name (confirmed in existing files), so the same fix applies with the updated package name.

The change is **trivial** — literally 5 lines added to `packages/core/tsconfig.json`.

**NEW content:**
```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "outDir": "dist",
    "tsBuildInfoFile": ".tsbuildinfo",
    "lib": ["DOM", "DOM.Iterable", "ES2021"],
    "composite": true,
    "types": ["node", "vitest/globals"],
    "baseUrl": ".",
    "paths": {
      "@vybestack/llxprt-code-core": ["./index.ts"]
    }
  },
  "include": ["index.ts", "src/**/*.ts", "src/**/*.json"],
  "exclude": ["node_modules", "dist", "**/*.test.ts", "**/*.spec.ts"]
}
```

**Explanation:**
- `baseUrl: "."` — resolves relative to the `packages/core/` directory
- `paths: { "@vybestack/llxprt-code-core": ["./index.ts"] }` — maps the package name to the local entry point

**Note:** The upstream diff uses ES2023, but LLxprt currently uses ES2021 in the file I read. Keep the existing ES version unless there's a reason to upgrade.

## Implementation Steps

1. **Edit `packages/core/tsconfig.json`:**
   - Add `"baseUrl": "."` to `compilerOptions`
   - Add `"paths": { "@vybestack/llxprt-code-core": ["./index.ts"] }` to `compilerOptions`

2. **Verify no build errors:**
   - Run `npm run typecheck` (or `tsc --noEmit` in `packages/core/`)
   - Should resolve any circular dependency warnings

3. **No tests needed** — this is a pure build configuration change
