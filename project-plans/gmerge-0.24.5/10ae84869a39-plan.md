# Playbook: Console → coreEvents/debugLogger Migration

**Upstream SHA:** `10ae84869a394a6214f26e812539f0285af6e72f`
**Upstream Subject:** Migrate console to coreEvents.emitFeedback or debugLogger
**Upstream Stats:** 66 files, 564 insertions, 425 deletions

## What Upstream Does

This commit is a **mechanical refactoring** that removes direct `console.*` usage throughout the codebase and replaces it with two proper abstractions:
- **`debugLogger`** for developer-facing debug/diagnostic messages
- **`coreEvents.emitFeedback(severity, message)`** for user-facing errors/warnings

It also adds an ESLint rule `'no-console': 'error'` to prevent future violations and documents the logging guidelines.

## LLxprt Adaptation Strategy

This is a **MECHANICAL PATTERN-BASED REFACTORING** requiring comprehensive file-by-file migration. From ripgrep analysis, LLxprt has **893 console.* violations** across packages/core and packages/cli that need classification and migration.

### Logging Decision Tree

For each `console.*` call, determine the correct replacement:

1. **Test files** (`*.test.ts`, `*.test.tsx`):
   - Mock setup (`console.error = vi.fn()`) → Keep if testing console output, otherwise replace with appropriate mock
   - Test output (`console.log('Test result')`) → Replace with `debugLogger.debug()` or remove
   - Expected errors being tested → May need `vi.spyOn(coreEvents, 'emitFeedback')` instead

2. **User-facing errors** (CLI, UI, failures):
   - `console.error('message')` → `coreEvents.emitFeedback('error', 'message')`
   - Multi-line error logs → Concatenate into single `emitFeedback()` call

3. **Developer diagnostics** (internal logging, debug info):
   - `console.log(...)` → `debugLogger.log(...)`
   - `console.warn(...)` → `debugLogger.warn(...)`
   - `console.error(...)` → `debugLogger.error(...)`
   - `console.debug(...)` → `debugLogger.debug(...)`

4. **Special cases**:
   - ConsolePatcher (UI redirection) → Keep as-is (intentionally wraps console)
   - Comments mentioning console → Keep unless outdated
   - Code examples in docs → Keep

**File:** `packages/cli/src/gemini.tsx`

Add UserFeedback listener in `initializeOutputListenersAndFlush()`:

```typescript
coreEvents.on(CoreEvent.UserFeedback, (payload: FeedbackPayload) => {
  const stream = payload.severity === 'error' ? process.stderr : process.stdout;
  stream.write(payload.message + '\n');
  if (payload.error) {
    stream.write('Error: ' + String(payload.error) + '\n');
  }
});
```
