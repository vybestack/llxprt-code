## What Upstream Does

Enables SessionStart hooks to inject additional context into the session via `hookSpecificOutput.additionalContext`. In non-interactive mode, context is prepended to the input prompt. In interactive mode, context is added to chat history as a user message. The hook also returns a `systemMessage` that's displayed to the user. The trigger function `fireSessionStartHook()` now returns `DefaultHookOutput | undefined` instead of `void`. CLI entry points (gemini.tsx, AppContainer.tsx, clearCommand.ts) consume the result and inject context accordingly.

## LLxprt Adaptation Strategy

LLxprt likely has similar SessionStart hook infrastructure but needs to wire up context injection:

1. **Hook trigger**: Modify `fireSessionStartHook()` to return `DefaultHookOutput | undefined`
2. **SessionStart output**: Ensure `SessionStartHookOutput` has `getAdditionalContext()` method
3. **Non-interactive CLI**: After SessionStart, prepend additionalContext to input if present, display systemMessage
4. **Interactive UI**: After SessionStart, add additionalContext to history, display systemMessage
5. **Clear command**: Consume SessionStart output after clearing, inject context
6. **Content generator**: Wrap FakeContentGenerator in LoggingContentGenerator for telemetry
7. **Tests**: Add integration tests for context injection (optional if test infrastructure exists)

## Implementation Steps

1. **Modify `sessionHookTriggers.ts`**:
   - Change `fireSessionStartHook()` return type from `Promise<void>` to `Promise<DefaultHookOutput | undefined>`
   - After receiving response, check `response.output`, call `createHookOutput('SessionStart', response.output)`, return it
   - On error or no output, return undefined

2. **Modify non-interactive CLI entry (e.g., `gemini.tsx` or `nonInteractiveCli.ts`)**:
   - Move stdin reading BEFORE SessionStart hook (so we can prepend context)
   - Call `fireSessionStartHook()` and capture result
   - If result has `systemMessage`, write to stderr
   - If result has `getAdditionalContext()`, prepend to input: `input = context + '\n\n' + input`

3. **Modify interactive UI entry (`AppContainer.tsx` or similar)**:
   - Call `fireSessionStartHook()` and capture result
   - If result has `systemMessage`, add to history as INFO message
   - If result has `getAdditionalContext()`, call `geminiClient.addHistory({ role: 'user', parts: [{ text: context }] })`

4. **Modify `clearCommand.ts`**:
   - Capture result from `fireSessionStartHook()`
   - If result has `systemMessage`, add to history as INFO message

5. **Modify `contentGenerator.ts`**:
   - In `createContentGenerator()`, wrap FakeContentGenerator: `new LoggingContentGenerator(await FakeContentGenerator.fromFile(...), gcConfig)`

6. **Add integration tests** (optional, only if test infrastructure supports it):
   - Test SessionStart with additionalContext → verify context appears in API request
   - Test SessionStart with systemMessage → verify message is displayed
   - Mock hook script that returns both fields

7. **Verify**: `npm run typecheck && npm run test -- packages/core/src/core/ packages/cli/src/`
