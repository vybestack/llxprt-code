## Remediation (P05 Verifier Failure Fix)

The P05 verifier found that the root export identity command failed for three symbols: `PROVIDER_ACCOUNTS_FILENAME`, `OAUTH_FILE`, and `FileSystemService`. Root cause and fix for each:

### `PROVIDER_ACCOUNTS_FILENAME` and `OAUTH_FILE`
- **Root cause**: Core barrel (`index.ts`) only exported `Storage` from `./config/storage.js` via `export { Storage }`, not `PROVIDER_ACCOUNTS_FILENAME` or `OAUTH_FILE`. The shim file (`config/storage.ts`) did `export *` from storage, but the barrel picked only `Storage`.
- **Fix**: Changed barrel export to `export { Storage, PROVIDER_ACCOUNTS_FILENAME, OAUTH_FILE } from './config/storage.js'`.

### `FileSystemService`
- **Root cause**: `FileSystemService` was a TypeScript interface, which is erased at runtime. `core.FileSystemService` evaluated to `undefined` at runtime, making strict identity comparison fail.
- **Fix**: Converted `FileSystemService` from an interface to an abstract class in `packages/storage/src/services/fileSystemService.ts`. This preserves the contract (abstract methods) while making the symbol a runtime value. All consumers use `implements FileSystemService` which works identically with abstract classes. Verified: storage typecheck, core typecheck, CLI typecheck, storage tests (191 pass), core tests (7609 pass), CLI fileSystemService tests (8 pass).
