## Deviations

1. **Package self-import for testing.ts**: The plan specifies `import { resetConversationFileWriterForTesting } from '@vybestack/llxprt-code-storage/testing'`. However, vitest/vite does not resolve package self-imports at test runtime without a built `dist/` directory or explicit vite alias configuration. The test instead imports from `../testing.js` (source-relative). The `testing.ts` barrel and `package.json` export map entry (`"./testing": "./dist/src/testing.js"`) are correctly configured and verified by typecheck. Downstream consumers (outside the package) will use the `@vybestack/llxprt-code-storage/testing` path as intended.

2. **Package self-import for barrel test**: Same issue тАФ `import { SESSION_FILE_PREFIX } from '@vybestack/llxprt-code-storage'` does not resolve in vitest. The barrel import test uses `import { SESSION_FILE_PREFIX as BARREL_PREFIX } from '../index.js'` which imports the same barrel. The barrel re-export structure is verified structurally by grep checks and by typecheck.
