```json

"format:check": "prettier --check ."
```
If `npm run format:check` exits non-zero for any reason other than formatting drift, investigate the root cause before proceeding.

The format verification must prove formatting is deterministic — running format twice produces no additional changes. **If any formatting drift is detected that cannot be resolved by two format runs, the phase FAILS.**

```bash
# Step 1: Run format to apply any formatting fixes
npm run format

# Step 2: Run format AGAIN to verify determinism (idempotency check)
npm run format

# Step 3: Verify no formatting drift remains — this MUST pass or the phase FAILS
npm run format:check
# format:check exits non-zero if any file has formatting drift.
# If this fails, the phase is FAIL — investigate and fix before proceeding.
# No command should succeed while merely printing changed files.

# Step 4: Review full diff for correctness (DO NOT stage or commit)

git diff HEAD -- '*.ts' ':!*.d.ts' ':!dist/**'
```

**CRITICAL**: Verification phase does NOT stage or commit changes. Do NOT run `git add`, `git add -A`, `git commit`, or any staging commands during any verification phase (P01-V through P07-V).

1. Core moved files are pure re-export shims (no implementation left).
