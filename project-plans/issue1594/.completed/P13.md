## Defects fixed

### Defect 1 (CRITICAL) — unimported probe helpers → ReferenceError at GREEN
**Root cause:** `disposal.spec.ts` imported ONLY `{ captureProbe, type DisposalProbe }` but called nine other probe functions (`agentClientDisposed`, `schedulerDisposed`, `confirmationCoordinatorDisposed`, `messageBusSubscriptionCount`, `lspDisposed`, `extensionsDisposed`, `sessionLocksReleased`, `isAggregateDisposeError`, `aggregateErrors`). At RED this was masked because `dispose()` throws NYI before those lines run; at GREEN every disposal test would crash with `ReferenceError: <fn> is not defined`. Neither typecheck (spec files excluded via `tsconfig.json`) nor ESLint (`no-undef` off for TS) catches this.
**Fix:** Added all nine missing imports to the `from './helpers/disposalProbe.js'` import block (alphabetized, Prettier-stable). The `createRecordingSchedulerFactory` import was also added (needed by Defect 2).

### Defect 2 (CRITICAL) — scheduler/confirmationCoordinator rows not GREEN-reachable
**Root cause:** The `scheduler` and `confirmationCoordinator` rows used `buildAgent('plain-text.jsonl')` with NO injected `toolSchedulerFactory` and no tool turn. Per dispose.md (lines 40-47 + table note "Most agents have NO facade-held scheduler"), these facade-held resources are **CONDITIONAL T19** — they do NOT EXIST for a plain agent, so `schedulerDisposed`/`confirmationCoordinatorDisposed` return `false` permanently and `toBe(true)` could never pass at GREEN.
**Fix:** Both rows now build the agent WITH an injected RECORDING `toolSchedulerFactory` (from `fakeScheduler.ts`, the same one T19 uses: `createRecordingSchedulerFactory()`) AND drive a tool turn (`tool-call-then-answer.jsonl` + `respondToFirstConfirmation(agent, ToolConfirmationOutcome.ProceedOnce)`), mirroring `scheduler-factory.spec.ts`. The agent runs `drain(agent.stream('use a tool'))`, asserts a `tool-call` event occurred and `createdHandles.length >= 1`, THEN captures the probe and asserts the injected handle's real disposed flag flips true after `await agent.dispose()`. Test-body comments cite dispose.md lines 40-47 (scheduler) and 45-47 (coordinator) mapping the row to CONDITIONAL T19 teardown. The remaining rows (agentClient, messageBus, lsp, extensions, sessionLock, idempotent re-dispose) keep the plain agent (their resources are unconditional).

### Defect 3 (MAJOR) — messageBus probe reads non-existent surfaces
**Root cause:** `messageBusSubscriptionCount` in `disposalProbe.ts` read `bus['subscriptionCount']` (does not exist), `bus['listenerCount']` (exists but is a FUNCTION requiring a `MessageBusType` arg, so `typeof !== 'number'` → skipped), and `bus['subscriptions']` (does not exist) → returned `-1` on the real bus, so `toBe(0)` could never pass at GREEN and a leaked subscription was undetectable.
**Fix:** Rewrote the probe to read the REAL surface. The honest GREEN-reachable count = total active listeners across all event types on the underlying private `emitter`: `emitter.eventNames().reduce((n, name) => n + emitter.listenerCount(name), 0)`. A new cast-exempt `readEmitter(bus)` helper narrows the private `emitter` field to a typed `EmitterLike` interface (`eventNames()` / `listenerCount(name)`) with bound-method invocation (`call(candidate, ...)`) — NO `any`. P24 (dispose.md lines 50-52) unsubscribes recorded subs, driving this count to the post-dispose baseline. The old bogus reads (`subscriptionCount`, `subscriptions`) are gone. Documentation comment explains the mapping.
