## The error+usage Mutual-Exclusion Defect (FIXED)

`buildAgentResult` (`packages/agents/src/api/agentBootstrap.ts`) previously used
MUTUALLY-EXCLUSIVE early returns. An errored turn that ALSO carried usage metadata
silently DROPPED `usage` — breaking non-interactive parity, because runNonInteractive
needs BOTH `usage` (stats / `--output-format json`) AND `error` (non-zero exit) when a
turn errors after consuming tokens.

### Before

```ts
export function buildAgentResult(drained: DrainedResult): AgentResult {
  const result: AgentResult = {
    text: drained.text,
    toolCalls: drained.toolCalls,
    finishReason: drained.finishReason,
  };
  if (drained.error !== undefined) {
    return { ...result, error: drained.error as AgentResult['error'] }; // <-- returns; DROPS usage
  }
  if (drained.usage !== undefined) {
    return { ...result, usage: drained.usage as SessionStats };
  }
  return result;
}
```

### After

```ts
export function buildAgentResult(drained: DrainedResult): AgentResult {
  return {
    text: drained.text,
    toolCalls: drained.toolCalls,
    finishReason: drained.finishReason,
    ...(drained.error !== undefined
      ? { error: drained.error as AgentResult['error'] }
      : {}),
    ...(drained.usage !== undefined
      ? { usage: drained.usage as SessionStats }
      : {}),
  };
}
```

`error` and `usage` are now INDEPENDENT optional surfaces, spread conditionally
(`exactOptionalPropertyTypes`-safe — never set to `undefined`, never mutually
exclusive). Casts remain honest (the existing `as AgentResult['error']` /
`as SessionStats` narrowing pattern; no `as any`).

## finishReason Derivation (CONFIRMED CORRECT — no change)

`drainToResult` (`agentBootstrap.ts:336`) sets `finishReason = event.reason` from the
`done` event. `DoneReason` (`event-types.ts:20`) =
`'stop' | 'aborted' | 'max-turns' | 'context-overflow' | 'loop-detected' | 'error' | 'hook-stopped'`.
It DISTINGUISHES `'aborted'` vs `'error'`, so runNonInteractive can map distinct exit
codes (no exit-mapping ambiguity gap).

## runNonInteractive Mapping Feasibility

| AgentResult field | runNonInteractive use |
| --- | --- |
| `text` | `--output-format text` → stdout body |
| `toolCalls` | `--output-format json` → structured tool-call array |
| `finishReason` (DoneReason) | exit-code selection (`stop`→0; `error`→non-zero; `aborted`→SIGINT-style; `max-turns`/`context-overflow`/`loop-detected`/`hook-stopped` distinguishable) |
| `error` (AgentError) | non-zero exit + stderr message |
| `usage` (SessionStats) | stats / `--output-format json` token accounting |

A thin CLI wrapper can render text/json, split stdout/stderr, auto-answer tools (via
`onApproval`), and choose exit status — entirely from the public `AgentResult`, with no
deep imports.
