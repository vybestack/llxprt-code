## Remediation — AgentAuthControl aligned to authoritative auth contract

A verification gate found ONE blocking defect: `AgentAuthControl` did not match
the authoritative `agent.auth.*` contract in `overview.md` L941–958 /
`specification.md` §REQ-008 / `plan/18-impl-auth-keys.md`. The flattened
key/setter members (`setApiKey`, `setApiKeyFile`, `setKeyName`, the old
`keys(): readonly AuthKeyEntry[]`, the old `buckets()`/`switchBucket(name)`,
and the single-arg `setBaseUrl`) could not express the T18b call path
`agent.auth.keys.save(...)` (storing a named-key REFERENCE, not a raw secret).

### Changes (confined to `packages/agents/src/api/agent.ts`)

- **Rewrote `AgentAuthControl`** to match overview.md L941–958 exactly:
  `login(provider, {bucket?})`, `logout(provider, {bucket?, all?})`,
  `status(provider?): AuthStatus` (was `ProviderStatus` — now the
  string-literal union the contract names), `enableOAuth(provider)`,
  `disableOAuth(provider)`, `listBuckets(provider?): readonly AuthBucket[]`,
  `switchBucket(provider, bucket)`, `mcpLogin(server)` (unchanged), the nested
  `readonly keys: AgentAuthKeysControl`, and
  `setBaseUrl(baseUrl: string | null, {provider?})`.
- **Added `AgentAuthKeysControl`** (the nested `agent.auth.keys.*` namespace):
  `list(): Promise<readonly KeyInfo[]>`, `save(name, apiKey, {provider?})`,
  `use(name, {provider?})`, `delete(name, {provider?})`,
  `setRaw(apiKey: string | null, {provider?})`,
  `setKeyFile(path: string | null, {provider?})`. This is the call path T18b
  exercises (`agent.auth.keys.save` stores a reference, not a raw secret).
- **Added `KeyInfo`** public type (`{ name: string; provider?: string }`) — a
  reference to a named key, never the secret — placed near the other public
  auth shapes.
- **Removed `AuthKeyEntry`** (was `{ keyName?, keyFile?, baseUrl?, apiKey? }`):
  it leaked `apiKey` (a raw secret) and became unused after the flattening was
  replaced by the `keys` namespace. Confirmed zero remaining references across
  `packages/`.
- **Kept `AuthBucket`** (still returned by `listBuckets`).
- `login`/`logout`/`enableOAuth`/`disableOAuth`/`switchBucket` now take
  `provider` as a REQUIRED first param per the contract; `login`/`logout` also
  take their opts bags.
- No `any`, no type assertions (`as X`), explicit return types on every
  signature, `readonly`/`Readonly` throughout. Interface definitions only — no
  implementation. The single `@plan`/`@requirement` marker block at the top of
  the file is unchanged; no new marker blocks added. No other interface, file,
  or phase's work was touched.
