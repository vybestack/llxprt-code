## 2. Semantic Verification Checklist (MANDATORY)

### Q1 — All 19 AgentEvent variants present and discriminated on `type`?

**YES.**

- **Schema** (`event-schema.ts`): `AgentEventSchema` is a genuine
  `z.discriminatedUnion('type', [ ... ])` with exactly 19 members, each an
  `z.object({ type: z.literal('<variant>'), ... })`. The discriminant key string is
  literally `'type'`. Confirmed it is a real `z.discriminatedUnion`, not a plain
  `z.union`.
- **Types** (`event-types.ts`): `AgentEvent` is a 19-arm TS union of `Readonly<{ type:
  '<variant>'; ... }>` objects, every arm carrying a `type` string-literal property —
  a proper TS discriminated union.
- The 19 variants line up 1:1 in both files: text, thinking, tool-call, tool-result,
  tool-confirmation, tool-status, usage, model-info, notice, compression,
  context-warning, retry, citation, loop-detected, idle-timeout, invalid-stream,
  hook-blocked, error, done.

### Q2 — DoneReason's 7 values present?

**YES.** Both files carry exactly the 7 values: `stop`, `aborted`, `max-turns`,
`context-overflow`, `loop-detected`, `error`, `hook-stopped`.

- Types: `export type DoneReason = 'stop' | 'aborted' | 'max-turns' |
  'context-overflow' | 'loop-detected' | 'error' | 'hook-stopped';`
- Schema: `DoneReasonSchema = z.enum([...7 values...])` and `AgentDoneEvent.reason`
  uses `DoneReasonSchema`.

### Q3 — Payload field names match specification.md §4.4 mapping table EXACTLY?

**YES — verified spot-by-spot against turn.ts and §4.4.**

- `text.text: string` — internal Content `value: string`. ✓

  `value: ChatCompressionInfo | null`; the public field is `ChatCompressionInfo | null`

- `idle-timeout.error: StructuredError` and `error.error: StructuredError` — internal

## 3. Schema-First Integrity Finding

- `AgentEventSchema` is a real `z.discriminatedUnion('type', [...19...])`.
- `DoneReasonSchema = z.enum([...7...])` covers exactly the 7 values and nothing more.
- Projection schemas exist for every projection type: `StructuredErrorSchema`,
  `ThoughtSummarySchema`, `UsageMetadataValueSchema`, `FinishedValueSchema`,
  `AgentStopInfoSchema`, `ModelInfoSchema`, `ChatCompressionInfoSchema`,
  `AgentToolCallSchema`, `AgentToolResultSchema`, `ToolConfirmationSchema`,
  `ToolUpdateStatusSchema`, `ToolUpdateSchema`.

**Drift-risk note (non-blocking, documented):** The TS `AgentEvent` union and the Zod
`AgentEventSchema` are authored as **two hand-maintained parallel definitions**; the
types are NOT derived via `z.infer` from the schema (nor is the schema derived from the
types). I manually diffed every one of the 19 arms field-by-field and they are
currently **fully consistent** (same variant set, same field names, same
optionality/nullability: `compression.info` nullable both sides; `done.finished`/
`done.stop` optional both sides; `context-warning` flat numbers both sides). Two
benign representational differences, both acceptable and not contract drift:
1. `ChatCompressionInfoSchema.compressionStatus` is `z.number()` while the TS
   `ChatCompressionInfo` (imported real type) uses the `CompressionStatus` enum — the
   enum is numeric, so the schema is a correct (if looser) runtime validator. The
   public TS type still carries the real enum.
2. `FinishedValue.reason` is `z.string()` / `string` in public, intentionally widened
   from the internal `FinishReason` enum (public decoupling) — consistent on both
   sides.

The residual risk is purely a **maintenance** one: future edits must touch both files.
This is within the spec's "rigorously in sync" allowance (specification.md says types
derived OR rigorously in sync). They are presently in sync. Recommend (non-blocking)
that a later phase add a type-level `Equals<AgentEvent, z.infer<typeof
AgentEventSchema>>` compile guard to lock the two together; not required to PASS P04a.

## 4. Holistic Functionality Assessment

**The public event surface (in my words):** `AgentEvent` is a stable, Zod-validated,
readonly discriminated union of 19 variants keyed on `type`, plus a 7-value
`DoneReason` enum, that fully decouples public consumers (#1595 CLI, scripts,
a2a) from the internal `ServerGeminiStreamEvent` / `GeminiEventType` (21 variants) and
`AgenticLoopEvent`. Payloads are projected to stable public shapes (AgentToolCall,
AgentToolResult, ToolConfirmation, ToolUpdate, AgentStopInfo, FinishedValue,
UsageMetadataValue) while ThoughtSummary/ModelInfo/ChatCompressionInfo/StructuredError
reuse the real core types (re-exported) rather than re-inventing them. Internal-only
fields (`prompt_id`, `responseParts`, `errorType`, `isClientInitiated`, `checkpoint`,
`hookRestrictedAllowedTools`, `agentId`, `outputFile`, `suppressDisplay`, Finished
`outcome`, `traceId`, the raw confirmation `request`) are intentionally dropped.
