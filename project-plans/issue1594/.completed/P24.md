with no modification to any spec or helper file.

## Step-80 remediation (real extension teardown via Config-owned ExtensionLoader)

The original step-80 block was a BARE FLAG-SET (`safe(() => { ownership.extensionsDisposed = true })`)
with no awaited teardown work — a pseudocode-compliance defect against
dispose.md line 80, which requires a REAL awaited extensions teardown. The
pseudocode's literal `ownership.extensionsManager.dispose()` does not exist in
the codebase, but the genuine Config-owned seam does:

- `ownership.config.getExtensionLoader(): ExtensionLoader`
  (core `configBaseCore.ts:588`) returns the real loader.
- `ExtensionLoader.getExtensions(): GeminiCLIExtension[]` (public) lists known
  extensions; `GeminiCLIExtension.isActive: boolean` distinguishes active ones.
- `ExtensionLoader.unloadExtension(ext): Promise<void>`
  (core `extensionLoader.ts:275`, the documented dynamic-unload path) stops the
  extension's MCP servers/context/custom-commands/subagents.

**Code change (before → after of the step-80 block in `agentImpl.ts`):**

Before:

    // @pseudocode dispose.md 80: NET-NEW extensions teardown. Config has no
    // extensionsManager field and the headless harness creates none, so this is
    // a safe no-op that still sets the completion marker AFTER the (no-op) await
    // (T13 observable).
    await this.safe(errors, () => {
      ownership.extensionsDisposed = true;
    });

After:

    // @plan:PLAN-20260617-COREAPI.P24 @requirement:REQ-016
    // @pseudocode dispose.md 80: NET-NEW extensions teardown. ... real
    // Config-owned ExtensionLoader ... unloadExtension ... marker AFTER the loop.
    const activeExtensions = collectActiveExtensions(
      ownership.config.getExtensionLoader(),
    );
    for (const extension of activeExtensions) {
      await this.safe(errors, () =>
        unloadExtensionSafely(ownership.config.getExtensionLoader(), extension),
      );
    }
    ownership.extensionsDisposed = true;

**Helpers added** (module-level, mirroring the `disposeOAuthManager`
runtime-guard idiom — structural optional-method narrowing, no `as any`):

- `collectActiveExtensions(loader)` — guards `getExtensions`, filters to
  `isActive`.
- `unloadExtensionSafely(loader, ext)` — guards `unloadExtension`, awaits it so a
  thrown unload propagates to the caller's `safe()` and is collected into
  `errors[]` (and thus into `AggregateDisposeError`).
- `ExtensionTeardownSurface` interface — the optional-method structural shape.
- Type-only import of `GeminiCLIExtension` from
  `@vybestack/llxprt-code-core/utils/extensionLoader.js`.

**Error-collection guarantee:** because each `unloadExtension` is wrapped in
`this.safe(errors, …)`, a failing unload is pushed into `errors[]` without
short-circuiting the remaining steps, and is surfaced at the end via
`throw new AggregateDisposeError(errors)` (dispose.md 100-102).

**Marker semantics preserved:** `ownership.extensionsDisposed = true` is set
AFTER the awaited loop (exactly like the sessionLocks pattern at lines 81-83);
in headless mode the loop is vacuously empty (zero active extensions) but the
step is now a GENUINE awaited teardown call-path rather than a bare flag-set.
