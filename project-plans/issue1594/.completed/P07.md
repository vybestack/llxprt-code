## Approach Chosen: Single-source (recommended)

`internals.ts` is the **single source** of the low-level re-export list. `index.ts` does `export * from './internals.js'` to preserve the existing top-level surface non-breakingly. This guarantees the top-level and the subpath expose the EXACT SAME low-level symbols and avoids duplication drift.

`createTaskToolRegistration` is intentionally NOT re-exported from `internals.ts` — it is app-glue (a factory function), not a power-user primitive, and re-exporting it would create a circular dependency (`internals.ts ← index.ts`). It remains exported solely from `index.ts`.

## Name Collisions Resolved

Two name collisions were detected by `npm run typecheck` when `export * from './internals.js'` and `export * from './api/index.js'` were both present in `index.ts`:

1. **`ApprovalHandler`** — exported by both barrels with DIFFERENT meanings:
   - Internals: `./core/agenticLoop/types.ts` (a turn-loop approval handler)
   - API: `./api/config-types.ts` (a user-facing confirmation callback)
2. **`CompressionResult`** — exported by both barrels with DIFFERENT meanings:
   - Internals: `@vybestack/llxprt-code-core/core/compression/types.js` (via `./compression/index.js`)
   - API: `./api/agent.ts` (a public API result type with status/token counts)

**Resolution:** Added explicit `export type { ... }` re-exports in `index.ts` from the low-level (internals) sources. In TypeScript, an explicit named re-export takes precedence over ambiguous `export *` merges, which (a) resolves the TS2308 ambiguity error and (b) preserves the existing low-level type definitions at the top level (non-breaking — no existing consumer's type changes).

**Note:** `ModelInfo`, `ChatCompressionInfo`, and `StructuredError` are also re-exported by both barrels, but both paths trace back to the SAME source module (`@vybestack/llxprt-code-core/core/turn.js`), so TypeScript deduplicates them automatically — no disambiguation needed.

## Non-breaking Proof

**Method:** Compared `git show HEAD:packages/agents/src/index.ts` (original) against the new `internals.ts` + `index.ts`.

**Result:** Every low-level `export` statement that existed in HEAD's `index.ts` was moved verbatim into `internals.ts` (identical source modules, identical named/type/star re-exports). `index.ts` re-exports all of them via `export * from './internals.js'`, plus retains `createTaskToolRegistration` (defined inline). **Zero top-level symbols were removed.** The only additions are the new public Agent API symbols (from `./api/index.js`) and the two explicit disambiguation re-exports.

**Consumer verification:** No monorepo consumer (cli, a2a-server) imports `ApprovalHandler` or `CompressionResult` from the agents top-level, confirming the disambiguation choice has no impact on existing imports.
