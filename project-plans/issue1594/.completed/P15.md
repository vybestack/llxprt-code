## Summary

The createAgent bootstrap (`createAgent.ts` / `agentImpl.ts` / `agentBootstrap.ts`
/ `loop/rebuildLoop.ts`) was already implemented in prior phases. P15 surfaced
one latent shared-core bug in `StreamProcessor._buildRuntimeContext` that was
blocking the target tests (T1 stream, T1 chat, T25a-d) from going GREEN. The
object-spread `{ ...baseRuntimeContext.config, ...params.config, tools }`
produced a plain object, stripping the live `Config` class prototype so
`config.getModel()` (called by `ProviderManager.resolveModelField`) threw
`TypeError: config.getModel is not a function`. CLI/a2a never hit it because
their model resolves earlier in `resolveModelField`, never reaching the
`config.getModel()` fallback branch; the fake/isolated bootstrap falls through
to that branch, exposing the bug.

This phase applied the precise prototype-stripping fix, verified the P15 target
tests go GREEN, ran the full verification suite (zero regressions), and created
this marker.
