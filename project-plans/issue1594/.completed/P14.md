## Key design decisions

1. **sawActivity gate (lines 30a-30g + 201):** Implemented exactly per
   pseudocode. A standalone stream-wrapped `AgentExecutionBlocked` does NOT set
   `sawActivity`; every other consumed loop event does. The loop-end
   synthesized `done` fires only when `NOT emittedDone AND (sawActivity OR
   pendingDoneReason != null)`. This satisfies both the NON-terminal Blocked
   row (0 done) and the real-abort row (Content seen → 1 done{stop}).

2. **mapFinishReason:** Returns `'stop'` unconditionally for Finished events
   (normal completion). Per the requirement, error/abort/etc. arrive via
   their own variants and set `pendingDoneReason`.

3. **toStopInfo:** Reads FLAT fields (`e.reason`, `e.systemMessage?`,
   `e.contextCleared?`) directly off the AgentExecutionStopped/Blocked
   events — never `e.value` (these variants carry NO `.value` wrapper).

4. **mapStreamEvent structure:** Split into `mapValueEvent` (value-bearing
   variants dispatched via switch with default) + early-return guards for
   no-value/terminal variants, keeping each function under the 80-line
   eslint limit while preserving the exact 21-variant semantics.

5. **config adapter structure:** Refactored `toConfigParameters` into
   `applySimpleMappings` / `applyArrayMappings` / `applyObjectMappings` /
   `applyCompoundMappings` / `applySettings` helpers with a declarative
   `SIMPLE_MAPPINGS` table to satisfy the function-length/complexity eslint
   rules. `modelParams` and `streamIdleTimeoutMs` are intentionally absent
   (no genuine ConfigParameters target field).

6. **systemPrompt/memory:** systemPrompt wins when both target `userMemory`
   (explicit, higher-precedence directive); documented with a one-line comment.

## Deviations from pseudocode

- **eventAdapter.ts structural split:** The pseudocode presents `mapStreamEvent`
  as a single 21-case switch. ESLint `max-lines-per-function` (80) and
  `sonarjs/nested-control-flow` forced a split into `mapValueEvent` (value
  variants) + early-return guards (no-value/terminal variants). The 21-variant
  semantics, state mutations, and ordering are identical to the pseudocode.
- **agentConfig.adapter.ts helper extraction:** The pseudocode's monolithic
  `toConfigParameters` (steps 10-101) exceeded `max-lines-per-function`,
  `complexity`, and `sonarjs/cognitive-complexity` limits. Extracted the field
  mappings into focused helpers and a declarative table. Field coverage and
  the settings-escape-hatch throw are unchanged.
- **default cases added:** ESLint `default-case` required a `default: return;`
  in the switch statements (no-op, since all variants are handled).
