## Types: reused vs newly projected

Reused clean public types (imported by type from their real, dependency-legal paths
under `@vybestack/llxprt-code-core`, which is a declared `agents` dependency):
- `ThoughtSummary` ← `@vybestack/llxprt-code-core/utils/thoughtUtils.js`
  (`{subject, description}`).
- `ModelInfo`, `ChatCompressionInfo`, `StructuredError` ←
  `@vybestack/llxprt-code-core/core/turn.js`.

Newly defined public projections (no clean public source existed; intentionally drop
internal fields):
- `DoneReason` — 7 stable string literals (adapter maps `FinishReason` enum to these).
- `UsageMetadataValue` — projection of the anonymous
  `ServerGeminiUsageMetadataEvent.value`; keeps `promptTokenCount?`,
  `candidatesTokenCount?`, `totalTokenCount?`, `cachedContentTokenCount?`.
- `FinishedValue` — projection of `ServerGeminiFinishedEvent.value`; keeps `reason`
  (string), `usageMetadata?`; DROPS `outcome?` (internal `hadVisibleOutput` /
  `hadThinking` / `hadToolCalls` impl detail).
- `AgentStopInfo` — projection of the FLAT `AgentExecutionStopped` / `AgentExecutionBlocked`
  events; keeps `reason`, `systemMessage?`, `contextCleared?`.
- `AgentToolCall` — projection of `ToolCallRequestInfo`; keeps `id`, `name`, `args`;
  DROPS `isClientInitiated`, `prompt_id`, `agentId`, `checkpoint`,
  `hookRestrictedAllowedTools`.
- `AgentToolResult` — projection of `ToolCallResponseInfo` / `CompletedToolCall`;
  keeps `id`, `name`, `output?` (display), `isError?`; DROPS `responseParts`,
  `errorType`, `agentId`, `outputFile`, `suppressDisplay`, raw `error`.
- `ToolConfirmation` — projection of `ServerToolCallConfirmationDetails`; carries BOTH
  `confirmationId` (correlation key for `respondToConfirmation`) and `toolCallId` (UI
  grouping) plus `name`, `details`; DROPS the raw `request` (`prompt_id`,
  `hookRestrictedAllowedTools`, etc.).
- `ToolUpdate` / `ToolUpdateStatus` — projection of scheduler `ToolCall`; keeps `id`,
  `name`, `status`, `output?`, `agentId?`; no raw scheduler union / CLI UI state.
