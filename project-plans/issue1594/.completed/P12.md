## Honest Scope Decisions

1. **T4f (stripThoughts normalization):** The FakeProvider replay fixture does not model per-provider thinking-block compatibility. The test asserts the behavioral contract via `agent.setHistory(history, { stripThoughts: true })` followed by `getHistory()` — asserting that thinking blocks are absent from the normalized history. The actual "switch INTO an incompatible provider" detection (which triggers stripThoughts automatically) is asserted through the `stripThoughts: true` option at the public API level; the per-provider compatibility detection itself is an impl-phase (P16) concern.

2. **T18 precedence chain:** FakeProvider does not model real key-resolution from environment/keyring. Each precedence tier is asserted via `agent.auth.keys.save()` / `agent.auth.keys.setRaw()` / `agent.auth.keys.setKeyFile()` + `agent.getProviderStatus().keyName` reflecting the winner. The full env-var tier is seeded via the harness config `auth` field rather than a real `process.env` mutation. The CHAIN ORDER (raw > CLI keyName > profile authKeyName > inline > keyfile > env) is asserted by applying tiers in sequence and observing the status change at each step.

3. **T18c OAuth/buckets:** The `onOAuthPrompt` handler is provided via `AgentConfig.onOAuthPrompt`. The test asserts that when a handler returns `true`, `auth.login()` succeeds (no throw) and `auth.status()` reflects `authenticated`; when no handler is registered, `auth.login()` rejects. Real OAuth token exchange is not modeled — the assertion is on the handler-driven accept/reject gate, which is the public surface.

4. **T20 discovery failure:** The fake MCP server's `failDiscovery()` causes the Agent's discovery path to yield `mcp_discovery_failed`. The `code` is extracted from the error event payload (`errors[0].error.code`). Since the `AgentError` type is structural and the error event's `error` field is typed as `StructuredError`, the code extraction uses a minimal `as unknown as { code?: string } | undefined` narrowing (cast-free in the assertion path via optional chaining).

5. **T25 provider bootstrap:** The "shared runtimeId/SettingsService" behavior is asserted by building two agents with distinct configs and asserting they are independent instances (not sharing mutable state). The "shared runtimeId" invariant is a P15 createAgent wiring concern; at RED the test asserts the public observable (distinct agents, distinct providers) which is the behavioral surface the CLI consumes.

6. **captureHistoryServiceIdentity:** At RED this returns `undefined` because the `AgentImpl` stub has no underlying HistoryService wiring. The T4d/T4e tests are structured so the switch call (`await agent.setProvider(...)` / `await agent.profiles.apply(...)`) — which throws NYI at RED — runs BEFORE the identity compare, so RED fails naturally at the switch. At GREEN (P15/P16) the helper reaches the real shared HistoryService and the identity assertion (`toBe`) becomes meaningful.
