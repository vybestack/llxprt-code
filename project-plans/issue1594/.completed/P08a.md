**Judgment: ACCEPTABLE — necessary, minimal, and correctly scoped.**

- **(a) NECESSARY:** The new node script `verify-api-property-ratio.js` uses node
  globals (`process`, `console`) and node-style patterns. The repo's base
  `no-restricted-syntax` rule (eslint.config.js:173) bans `CallExpression[callee.name="require"]`
  and string-literal throws; without this block the script would not be guaranteed
  lint-clean as a directly-run node script. The targeted `npx eslint` run proves the
  block is what makes the file pass (0 errors).
- **(b) MINIMAL + CORRECTLY SCOPED:** `files` matches ONLY
  `packages/agents/scripts/**/*.js`. It does NOT touch `src/**`, does not relax any rule
  for production code, and only turns off `no-restricted-syntax` plus adds node globals.
  Notably it is an EXACT MIRROR of the pre-existing, already-accepted block for
  `packages/vscode-ide-companion/scripts/**/*.js` (eslint.config.js:740-753) — and is
  even more conservative (it does not also disable `@typescript-eslint/no-require-imports`).
- **(c) HARMLESS:** No effect on the rest of the repo; full-package lint shows 0 errors
  and no new warnings.

This is best characterized as an **in-scope-but-unlisted dependency** of the new script
being lint-clean: although Phase 08's "Files to Modify" listed only
`packages/agents/package.json`, a lint-clean node script inherently requires the lint
config to recognize it. This does NOT warrant a FAIL.
